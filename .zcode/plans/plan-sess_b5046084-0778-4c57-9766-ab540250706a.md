# 启用 hao-code tokimo 沙箱功能 — 实施计划

工作区：`/Users/wanghao/git/hao-work/.worktrees/hao-code-sandbox-research`（分支 `hao-code-sandbox-research`）
SDK 版本：sk-wang/hao-code v1.14.0（已确认 `app/Sdk/Sandbox/` 与锁定版完全一致）

## 核心设计原则

1. **严格对称 `_fe_hitlMode` 链路**：opencode.json `_fe_sandbox*` → server state.config merge → worker request.sandbox → worker.php `normalizeSandbox()` → `HaoCodeConfig::sandbox`。
2. **不动 SDK**：所有 sandbox 配置通过 `SandboxConfig::tokimo(...)` 工厂方法入参，不依赖 SDK 未公开的 env 变量。
3. **不动 permissionMode**：`bypass_permissions` 保留。SDK 启用 sandbox 后 `toolFilter()` 自动接管工具路由（HaoCodeConfig.php:511-537），bypass 只跳过 ask，二者正交。
4. **错误通道复用**：sandbox 启动失败抛 RuntimeException，worker.php:142-149 已有 catch → `_fe_exception` 透出，前端按现有错误处理。
5. **沙箱 ≠ 工作目录沙箱**：`_fe_sandbox*` 是 VM 执行沙箱配置；现有 `project.sandboxes`（git worktree 列表）和 `source: 'sandbox'`（HITL auto_decision 枚举）含义不同，命名要避免混淆（前者保留原名，后者是 SDK 自动产出，无需改）。

## 数据流

```
[opencode.json]
  _fe_sandboxEnabled: true
  _fe_sandboxBaseRootfs: '/Users/.../haocode-sandbox/base'  ← UI"准备沙箱"按钮下载后写入
  _fe_sandboxNetwork: 'blocked'
  _fe_sandboxMemoryMb: 4096
  _fe_sandboxCpuCount: 4
        │ (PATCH /config merge 到 state.config,兼容现有 _fe_hitlMode 同样的无白名单 merge)
        ▼
[compat-server.js runSession]
  sandboxConfig = readSandboxConfig(state.config, session.directory, env)
  supervisor.run({ request: { ..., sandbox: sandboxConfig } })
        │ (worker-supervisor.js:31 spawn,HAOCODE_SANDBOX_BINARY env 透传)
        ▼
[worker.php]
  $sandbox = normalizeSandbox($request['sandbox'])  // null 或 SandboxConfig::tokimo(...)
  new HaoCodeConfig(..., sandbox: $sandbox)
        │
        ▼
[SDK SdkRunFactory.php:36-37]
  SandboxManager::create($config->sandbox, $config->cwd)
  → TokimoSandboxBackend 启动 VM (binary 路径由 options.binary 决定)

        ▼
[smart-HITL 联动]
  SDK SmartInterruptDecider 给沙箱内动作打 source:'sandbox'
  → AutoDecisionCard.tsx 渲染"沙箱"徽章(已有死代码,激活)
```

### 改动 1.1: `packages/haocode-bridge/worker.php`

在 `normalizeInterrupts`（worker.php:258-281）后新增 `normalizeSandbox`：

```php
/**
 * Build a SandboxConfig from the worker request, or null when sandbox is
 * disabled. Mirrors normalizeHitlMode's "whitelist + fail-closed" style:
 * any invalid field falls back to a safe default; only baseRootfs/binary
 * path issues throw (caught by the outer try/catch as _fe_exception).
 *
 * @return \HaoCode\Sdk\Sandbox\SandboxConfig|null
 */
function normalizeSandbox(mixed $value): ?object
{
    if (! is_array($value) || ($value['enabled'] ?? false) === false) {
        return null;
    }
    $provider = normalizeString($value['provider'] ?? null) ?? 'tokimo';
    if ($provider !== 'tokimo') {
        // MVP only supports tokimo; unknown providers are disabled.
        return null;
    }
    $baseRootfs = normalizeString($value['baseRootfs'] ?? null);
    if ($baseRootfs === null) {
        throw new InvalidArgumentException('tokimo sandbox requires baseRootfs.');
    }
    $network = normalizeString($value['network'] ?? null);
    if (! in_array($network, ['blocked', 'allow-all'], true)) {
        $network = 'blocked';
    }
    return \HaoCode\Sdk\Sandbox\SandboxConfig::tokimo(
        baseRootfs: $baseRootfs,
        mode: normalizeString($value['mode'] ?? null) ?? 'full',
        sync: normalizeString($value['sync'] ?? null) ?? 'upload-cwd',
        remoteCwd: normalizeString($value['remoteCwd'] ?? null) ?? '/workspace',
        cleanup: normalizeString($value['cleanup'] ?? null) ?? 'always',
        root: normalizeString($value['root'] ?? null),
        exclude: is_array($value['exclude'] ?? null) ? $value['exclude'] : [],
        binary: normalizeString($value['binary'] ?? null),
        vmDir: normalizeString($value['vmDir'] ?? null),
        memoryMb: normalizePositiveInt($value['memoryMb'] ?? null) ?? 4096,
        cpuCount: normalizePositiveInt($value['cpuCount'] ?? null) ?? 4,
        network: $network,
        startupTimeoutSeconds: normalizePositiveInt($value['startupTimeoutSeconds'] ?? null) ?? 30,
    );
}
```

在 `HaoCodeConfig` 构造（worker.php:85-107）的 `oauthBearer` 行后加：

```js
const SANDBOX_NETWORKS = new Set(['blocked', 'allow-all']);
const normalizeSandboxNetwork = (v) => SANDBOX_NETWORKS.has(v) ? v : 'blocked';
const DEFAULT_SANDBOX_MEMORY_MB = 4096;
const DEFAULT_SANDBOX_CPU_COUNT = 4;

/**
 * Build the worker `sandbox` request field from project config + runtime env.
 * Returns {enabled:false} when sandbox is off (worker normalizes to null).
 */
const buildSandboxRequest = (sandboxConfig, env) => {
  if (!sandboxConfig || sandboxConfig._fe_sandboxEnabled !== true) {
    return { enabled: false };
  }
  const baseRootfs = sandboxConfig._fe_sandboxBaseRootfs;
  if (!baseRootfs || typeof baseRootfs !== 'string') {
    return { enabled: false };  // Not provisioned yet; fail closed.
  }
  return {
    enabled: true,
    provider: 'tokimo',
    baseRootfs,
    binary: env.HAOCODE_SANDBOX_BINARY || undefined,  // Electron injects this
    network: normalizeSandboxNetwork(sandboxConfig._fe_sandboxNetwork),
    memoryMb: Number.isFinite(sandboxConfig._fe_sandboxMemoryMb) ? sandboxConfig._fe_sandboxMemoryMb : DEFAULT_SANDBOX_MEMORY_MB,
    cpuCount: Number.isFinite(sandboxConfig._fe_sandboxCpuCount) ? sandboxConfig._fe_sandboxCpuCount : DEFAULT_SANDBOX_CPU_COUNT,
  };
};
```

### 改动 3.1: `packages/electron/scripts/prepare-haocode-runtime.mjs`

新增 stage sandbox runner。在 `prepareBridge()` 之后加 `prepareSandboxRunner()`：

```js
const SANDBOX_RELEASE_TAG = 'v1.14.0';  // 与 composer.lock 中 sk-wang/hao-code 对齐

const sandboxAssetName = () => {
  const target = resolveTargetArchitecture();
  if (process.platform === 'darwin') return `haocode-sandbox-darwin-${target.node}`;  // 仅 arm64
  if (process.platform === 'linux')   return `haocode-sandbox-linux-${target.node}`;   // arm64/amd64
  if (process.platform === 'win32')   return null;  // Windows 用下方 svc + runner 双资产
  return null;
};

const sandboxAssets = () => {
  const target = resolveTargetArchitecture();
  if (process.platform === 'win32') {
    return ['haocode-sandbox-windows-amd64.exe', 'haocode-sandbox-svc-windows-amd64.exe'];
  }
  const name = sandboxAssetName();
  return name ? [name] : [];
};

const prepareSandboxRunner = async () => {
  const assets = sandboxAssets();
  if (assets.length === 0) return;  // 当前平台 SDK 不支持
  const dest = path.join(outputRoot, 'sandbox');
  fs.rmSync(dest, { recursive: true, force: true });
  fs.mkdirSync(dest, { recursive: true });
  for (const asset of assets) {
    const shaUrl = `https://github.com/skvdhshuk-blip/hao-code/releases/download/${SANDBOX_RELEASE_TAG}/${asset}.sha256`;
    const binUrl = `https://github.com/skvdhshuk-blip/hao-code/releases/download/${SANDBOX_RELEASE_TAG}/${asset}`;
    const expected = (await (await fetch(shaUrl)).text()).trim().split(/\s+/)[0];  // 形如 "b270826...  asset"
    const buffer = Buffer.from(await (await fetch(binUrl)).arrayBuffer());
    const actual = createHash('sha256').update(buffer).digest('hex');
    if (actual !== expected) throw new Error(`Sandbox binary ${asset} failed SHA256 verification.`);
    const dest2 = path.join(dest, asset);
    fs.writeFileSync(dest2, buffer, { mode: 0o755 });
    if (process.platform !== 'win32') fs.chmodSync(dest2, 0o755);
  }
};
```

## 风险与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| tokimo 在用户的 macOS 非 Apple Silicon / Windows 非 amd64 / 容器内不可用 | 沙箱启用失败 | worker.php catch RuntimeException → UI 显示明确错误；UI 检测 platform 不支持时禁用开关 |
| 首次 rootfs 下载失败（网络/磁盘） | 沙箱不可用 | `/sandbox/prepare` 重试机制 + 进度反馈；下载是幂等的，可重复触发 |
| Windows UAC 被拒 | svc 装不上 | UI 显示"管理员权限被拒绝，沙箱功能需要安装系统服务"+ 重试按钮 |
| baseRootfs 路径失效（用户清理缓存） | 已配 sandbox 启动失败 | server 启动时检测 baseRootfs 存在性，缺失则置空 + 提示重下 |
| 现有 release.yml 无 prepare:haocode-runtime（gap） | CI 产物不含 PHP+runner | PR3 必须显式补这一步，否则整个沙箱链断裂 |
